# V8 Gray S14 训练经验总结

本文记录 `v8_yolo11s_gray_s14_realboostr7_augmax_e500` 这条实验线从提出动机、配置设计、训练过程到最终效果的完整总结，重点回答两个问题：

- 我们在 `S12` 的基础上具体又做了什么
- 更强的数据增强到底带来了什么收益，又付出了什么代价

## 背景

在 `gray` 路线里，`S12` 已经是当时最稳的一条 `YOLO11s` 候选：

- full `valid/test mAP50-95(P)` 约为 `0.9812 / 0.9627`
- real `valid/test` 都做到 `misses=0`
- real `valid/test mean_pred_objects` 都是 `1.0`
- real `valid/test mean_norm_kp_err` 约为 `0.0101 / 0.0140`

但 `S12` 仍然留下了两个可以继续挖的点：

- `1_001860...` 这张顽固 hard case 虽然已经不再 miss，但关键点误差仍然是 `test` 子集里最大的残余项
- `S12` 的增强虽然已经不弱，但还没有把“更激进的非翻转增强”推到上限

所以 `S14` 的目标很明确：

- 不改变 `gray + YOLO11s + realboost_r7` 这条已经被验证有效的大方向
- 从 `S12 best.pt` 继续训，而不是回退到更早期 checkpoint
- 明确测试“把增强再往上推一档”是否还能继续改善 real 子集表现

## 我做了什么

`S14` 不是一条从头训练的新模型，而是一条从 `S12` 继续迁移的强化实验线。

起点和不变项：

- 起点 checkpoint:
  `runs/pose/v8_yolo11s_gray_s12_realboostr7_augopen_e500/weights/best.pt`
- 数据入口:
  `data.realboost_r7.yaml`
- 预处理:
  `preprocess.grayscale=true`
- 训练骨架仍然保持：
  - `epochs=500`
  - `patience=80`
  - `batch=4`
  - `imgsz=960`
  - `optimizer=AdamW`
  - `lrf=0.2`
  - `warmup_epochs=1`
  - `pose=36.0`

这次真正改动的核心只有两类：

### 1. 学习率略收

- `S12`: `lr0=1e-5`
- `S14`: `lr0=8e-6`

原因很简单：`S14` 是从一个已经很强的 `S12 best.pt` 出发，如果增强再变强而学习率不收，模型更容易被大增强扰动冲偏。

### 2. 非翻转增强继续上推

`S14` 相比 `S12` 的增强变化如下：

- `erasing`: `0.25 -> 0.35`
- `hsv_s`: `0.30 -> 0.35`
- `hsv_v`: `0.18 -> 0.22`
- `degrees`: `5.0 -> 7.0`
- `translate`: `0.10 -> 0.14`
- `scale`: `0.30 -> 0.38`
- `shear`: `3.0 -> 4.0`
- `perspective`: `0.001 -> 0.0015`
- `mosaic`: `0.70 -> 0.85`
- `mixup`: `0.15 -> 0.20`
- `copy_paste`: `0.15 -> 0.20`

不变的约束：

- `flipud=0.0`
- `fliplr=0.0`

也就是说，`S14` 的本质就是：

- 仍然坚持“不允许翻转”
- 但把光照、尺度、位移、形变、拼图和样本混合都进一步拉强

## 训练过程

`S14` 的训练不是那种“越往后越稳步上涨”的曲线，它更像是一条典型的“强增强长跑”曲线：

- best epoch 出现在 `232`
- 当时训练内最佳 `metrics/mAP50-95(P)=0.98147`
- 对应 `val/pose_loss=0.64975`
- 最后一轮是 `epoch 462`
- 最后一轮训练内 `metrics/mAP50-95(P)` 已经回落到 `0.97612`

这说明两件事：

- 这条线确实需要长跑，但最佳点出现得比 `S12` 更早
- 强增强版更依赖 `best.pt`，不能拿最后一轮代表模型最终水平

换句话说，`S14` 比 `S12` 更像一条“需要依赖早停和 best checkpoint 保底”的实验线。

## 最终效果

### Full valid/test

`post` 全量结果：

- full `valid mAP50-95(P) = 0.97547`
- full `test mAP50-95(P) = 0.97149`

### Real valid/test

我又单独对 real 子集做了复核，`conf=0.25` 下结果是：

- real `valid`
  - `misses=0`
  - `mean_pred_objects=1.0`
  - `max_pred_objects=1`
  - `mean_norm_kp_err=0.00927`
  - `mean_best_iou=0.86543`

- real `test`
  - `misses=0`
  - `mean_pred_objects=1.0`
  - `max_pred_objects=1`
  - `mean_norm_kp_err=0.01195`
  - `mean_best_iou=0.81793`

## 和 S12 对比

把 `S14` 和上一代最强灰度 `s` 模型 `S12` 放在一起看，会更清楚这条线到底学到了什么。

### 提升

- real `valid mean_norm_kp_err`
  - `S12: 0.01013`
  - `S14: 0.00927`

- real `test mean_norm_kp_err`
  - `S12: 0.01399`
  - `S14: 0.01195`

- full `test mAP50-95(P)`
  - `S12: 0.96268`
  - `S14: 0.97149`

- 对最顽固的 `1_001860...`
  - `S12` 的该样本关键点误差约 `0.0359`
  - `S14` 降到约 `0.0259`

这说明更强增强版确实把模型的真实场景鲁棒性再往前推了一步，特别是对 hardest test case 有明显帮助。

### 退步

- full `valid mAP50-95(P)`
  - `S12: 0.98119`
  - `S14: 0.97547`

- real `valid/test mean_best_iou` 都比 `S12` 更低
  - `S12` 更擅长给出“框更贴”的解
  - `S14` 更擅长给出“关键点更稳”的解

这说明 `S14` 并不是全面碾压 `S12`，而是发生了一个很清楚的 tradeoff：

- `S12` 更均衡、更稳、更像“默认主力模型”
- `S14` 更偏鲁棒性和 hard case 修复，尤其对真实 `test` 和难例关键点更友好

## 经验总结

从 `S14` 这条线里，我认为最重要的经验有 4 条：

### 1. `S12` 的方向是对的，继续沿它做“强增强微调”是值得的

如果底层路线不对，增强更强只会把模型打坏。但 `S14` 并没有崩，反而在 real 子集和 hardest case 上继续变好，这说明 `S12 -> 更强增强` 是一条可持续探索的方向。

### 2. 强增强对 real hard case 有真实收益

`S14` 最大的价值，不是 full `valid` 指标，而是它对真实场景中的困难样本表现更强，尤其是 `1_001860...` 这种此前一直是瓶颈的图。

### 3. 强增强版更依赖 `best.pt`

`S14` 的最佳点出现在 `epoch 232`，后面继续跑只是轻微回落。
所以这类实验不能只看最后一轮，必须用 `best.pt` 作为最终交付依据。

### 4. 更强增强会换来“关键点更稳，但框 IoU 略退”

这不是偶然噪声，而是很清晰的行为变化：

- `S14` 更偏向姿态鲁棒性
- `S12` 更偏向整体均衡和框贴合度

## 当前结论

- 如果目标是“灰度主线里最稳的默认 `YOLO11s`”，`S12` 仍然更像主力候选。
- 如果目标是“继续修 real hard case，尤其是最难的 `test` 图片”，`S14` 证明了更强的非翻转增强是有效方向。
- 后续如果继续沿这条线探索，最合理的不是回到 `S12` 前的旧配置，而是在 `S14` 附近继续做小步微调：
  - 可以保留更强增强的大框架
  - 重点试更温和的 `mosaic/mixup/copy_paste` 回调
  - 或只在 `valid` 侧轻微收一点增强，看看能否把 `full valid` 再拉回来
